# 测试用例分析器

## 功能说明

从手工测试用例中提取关键信息，生成结构化的自动化用例设计数据。

## 分析维度

### 1. 业务特征分析
- 功能模块识别
- 业务场景分类
- 测试类型判断（正向/异常/边界）
- 优先级提取

### 2. 配置参数分析
- 开关状态提取
- 配置项识别
- 接入方式识别

### 3. 数据依赖分析
- 前置数据
- 测试数据
- 环境依赖

### 4. 步骤模式分析
- 公共步骤识别
- 业务流程识别
- 验证步骤识别

## 使用方法

```python
from skills.case-skills.test-case-analyzer.test_case_analyzer import TestCaseAnalyzer

# 初始化分析器
analyzer = TestCaseAnalyzer()

# 加载手工用例数据
analyzer.load_from_file('sandbox/workspace/lark_latest_data.json')

# 执行分析
result = analyzer.analyze()

# 保存结果
analyzer.save_to_file('sandbox/workspace/test_case_analysis.json')

# 按配置状态分类
by_config = analyzer.get_cases_by_config()

# 按用户操作分类
by_operation = analyzer.get_cases_by_operation()

# 按测试类型分类
by_type = analyzer.get_cases_by_test_type()
```

## 应用场景

### 1. 目录结构设计
根据配置状态分类，设计合理的目录结构：
```
UNC认证业务/
├── 业务自动化-认证校验/
│   ├── 注册无感控制/
│   │   ├── 【开关状态】无感接入类型-仅有线/
│   │   ├── 【开关状态】无感接入类型-仅无线/
│   │   └── 【开关状态】无感接入类型-全部/
```

### 2. 用例分组
根据用户操作分类，合理组织用例：
```
有线认证场景/
├── TP-011: 历史数据未满足MAC快速认证数...（正向）
├── TP-012: 历史数据已满足MAC快速认证数...
└── ...
```

### 3. 公共步骤识别
识别可复用的步骤模式：
- 管理员登录（quote_id: 55635）
- 删除无感记录（quote_id: 65758）
- 有线认证流程（quote_id: 65650）

### 4. 数据驱动设计
提取测试数据变量：
- 用户账号
- 配置参数
- 环境变量

## 最佳实践

1. **分析先行**: 在创建自动化用例前，先进行结构化分析
2. **多维度分类**: 从多个维度分类用例，找到最优的组织方式
3. **识别复用**: 提取公共步骤和验证模式，提高复用率
4. **数据驱动**: 将配置参数和测试数据参数化，支持数据驱动

## 关键词提取规则

### 配置参数关键词
- `无感接入类型为仅有线` → 无感接入类型=仅有线
- `无感接入类型为仅无线` → 无感接入类型=仅无线
- `无感接入类型为全部` → 无感接入类型=全部
- `历史数据已满足MAC快速认证数` → MAC快速认证数=满足
- `历史数据未满足MAC快速认证数` → MAC快速认证数=未满足
- `接入方式未勾选MAC快速接入` → MAC快速接入=未勾选
- `MAC快速认证数为1/2` → MAC快速认证数=1/2

### 用户操作关键词
- `有线认证`、`有线portal认证` → 有线认证
- `无线认证`、`无线portal认证`、`无线1x认证` → 无线认证
- `MAB认证` → MAB认证
- `抢占MAC` → 抢占MAC

### 测试类型关键词
- `（正向）` → 正向测试
- 其他 → 常规测试
