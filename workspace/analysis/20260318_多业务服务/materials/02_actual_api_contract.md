## 🔍 业务服务管理API

### API 1: 新增业务服务

**端点**: `/sam/api/admin/businessServer/add`  
**方法**: POST  
**步骤类型**: 新增业务服务配置(非用户)

#### 请求示例
```json
{
  "authBusinessServer": true,
  "businessServerShort": "DYJ",
  "businessServerId": "打印机",
  "businessServerDes": "打印机服务"
}
```

#### 字段说明
| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| authBusinessServer | boolean | 是否启用认证 | true |
| businessServerShort | string | 业务服务简称 | "DYJ" |
| businessServerId | string | 业务服务ID | "打印机" |
| businessServerDes | string | 业务服务描述 | "打印机服务" |

#### 预期响应
```json
{
  "code": 0,
  "message": "成功",
  "data": {
    "businessServerUuid": "444840c89cdadf37019cdb8a27410dba",
    "businessServerId": "打印机"
  }
}
```

#### 业务服务ID枚举
- 打印机
- IP
- 邮箱

### API 2: 查询业务服务

**端点**: `/sam/api/admin/businessServer/query`  
**方法**: POST  
**步骤类型**: 查询业务服务配置

#### 请求示例
```json
{
  "pageIndex": 1,
  "businessServerId": "打印机",
  "pageSize": 10,
  "businessServerDes": ""
}
```

#### 字段说明
| 字段名 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| pageIndex | number | 页码 | 1 |
| businessServerId | string | 业务服务ID(可选) | "" |
| pageSize | number | 每页数量 | 10 |
| businessServerDes | string | 业务服务描述(可选) | "" |

#### 预期响应
```json
{
  "code": 0,
  "message": "成功",
  "data": {
    "total": 3,
    "list": [
      {
        "businessServerUuid": "444840c89cdadf37019cdb8a27410dba",
        "businessServerId": "打印机",
        "businessServerDes": "打印机服务",
        "authBusinessServer": true,
        "businessServerShort": "DYJ"
      }
    ]
  }
}
```

### API 3: 删除业务服务

**端点**: `/sam/api/admin/businessServer/delete`  
**方法**: POST  
**步骤类型**: 删除业务服务配置

#### 请求示例
```json
{
  "businessServerUuids": [
    "444840c89cdadf37019cdb8a27410dba"
  ]
}
```

#### 字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| businessServerUuids | array | 业务服务UUID列表 |

#### 预期响应
```json
{
  "code": 0,
  "message": "删除成功"
}
```

#### 错误场景
```json
{
  "code": 40001,
  "message": "删除失败!原因:业务服务正在使用中,请确保没有子业务服务用户使用业务服务!"
}
```

## 🎯 关键发现

### 业务范围差异

**初始理解**: 创建业务服务用户(开通账号给用户使用)  
**实际需求**: 管理业务服务配置(创建/删除打印机/IP/邮箱服务本身)

**差异说明**:
- ❌ 原设计: 以为是要创建业务服务用户的API
- ✅ 实际需要: 管理业务服务本身的API(服务配置)

### ⚠️ 重要补充：业务服务用户层面

经过重新分析手工用例，发现实际存在**两层**操作：

1. **业务服务配置层面** (已有API)
   - 新增业务服务 (打印机/IP/邮箱)
   - 查询业务服务
   - 删除业务服务

2. **业务服务用户层面** (⚠️ 缺少API信息)
   - 用户开户（通过第三方接口开通账户）
   - 用户销户（上网服务用户销户、业务服务用户销户）
   - 开通业务服务用户（给用户开通打印机/IP/邮箱服务）
   - 修改业务服务用户（编辑功能）
   - 业务服务用户解锁
   - 业务服务用户恢复
